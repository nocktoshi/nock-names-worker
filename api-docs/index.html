NockNames API

Resolve, search, and register .nock names.

A small JSON API backing NockNames. No auth. CORS enabled.

Example
curl 'https://api.nocknames.com/resolve?name=regen.nock'
Response
{
  "address": "8s29XU…TT5"
}
Example only — actual address depends on registry state.

Quickstart

The API responds with JSON and uses standard HTTP status codes. All endpoints support CORS.

Resolve a name
curl 'https://api.nocknames.com/resolve?name=regen.nock'

Endpoints

GET
/resolve

Resolve in either direction: name → address or address → name.

Query parameters
  • name (string) — Must match /^[a-z0-9]+\.nock$/.
  • address (string) — Blockchain address.
Provide exactly one of name or address.
Resolve name → address
curl 'https://api.nocknames.com/resolve?name=regen.nock'
200
{
  "address": "…"
}
404
{
  "error": "Name not registered"
}
Resolve address → name
curl 'https://api.nocknames.com/resolve?address=8s29XUK8Do7QWt2MHfPdd1gDSta6db4c3bQrxP1YdJNfXpL3WPzTT5'
200
{
  "name": "regen.nock"
}
400
{
  "error": "Missing address or name parameter"
}
GET
/search

Check availability/status for a name, or list registrations for an address.

Query parameters
  • name (string) — A label or full domain. Both regen and regen.nock are accepted.
  • address (string) — Blockchain address.
Search a name
curl 'https://api.nocknames.com/search?name=regen.nock'
200
{
  "name": "regen.nock",
  "price": 500,
  "status": "available | pending | registered",
  "owner": "… | null",
  "registeredAt": 1730000000000 | null
}
Pricing is based on the label length (see Pricing).
Search by address
curl 'https://api.nocknames.com/search?address=8s29XUK8Do7QWt2MHfPdd1gDSta6db4c3bQrxP1YdJNfXpL3WPzTT5'
200
{
  "address": "…",
  "pending": [
    { "address": "…", "name": "regen.nock", "status": "pending", "timestamp": 1730000000000 }
  ],
  "verified": [
    { "address": "…", "name": "regen.nock", "status": "registered", "timestamp": 1730000000000, "date": "…", "txHash": "…" }
  ]
}
GET
/pending

List pending registrations (most recent first).

curl 'https://api.nocknames.com/pending'
200
[
  { "address": "…", "name": "regen.nock", "status": "pending", "timestamp": 1730000000000 }
]
GET
/verified

List registered names (most recent first).

curl 'https://api.nocknames.com/verified'
200
[
  { "address": "…", "name": "regen.nock", "status": "registered", "timestamp": 1730000000000, "date": "…", "txHash": "…" }
]
POST
/register

Create a pending registration for a name and address.

Request
curl -X POST 'https://api.nocknames.com/register' \
  -H 'Content-Type: application/json' \
  -d '{"address":"8s29XU…TT5","name":"regen.nock"}'
Body
{
  "address": "string",
  "name": "lowercase alphanumeric + .nock"
}
200
{
  "address": "…",
  "name": "regen.nock",
  "status": "pending",
  "timestamp": 1730000000000
}
400
{ "error": "Name already registered | Invalid blockchain address | Name must be alphanumeric lowercase and end with .nock" }
POST
/verify

Verify payment and finalize registration. This checks for an unused payment transaction from the provided address that pays at least the required fee to the NockNames payment address.

Payment address
8s29XUK8Do7QWt2MHfPdd1gDSta6db4c3bQrxP1YdJNfXpL3WPzTT5
Pricing (label length)
  • len ≥ 10 → 100
  • len ≥ 5 → 500
  • len 1–4 → 5000
Request
curl -X POST 'https://api.nocknames.com/verify' \
  -H 'Content-Type: application/json' \
  -d '{"address":"8s29XU…TT5","name":"regen.nock"}'
200
{
  "message": "Registration successful!",
  "registration": {
    "address": "…",
    "name": "regen.nock",
    "status": "registered",
    "timestamp": 1730000000000,
    "date": "2025-01-01T00:00:00.000Z",
    "txHash": "…"
  }
}
400
{
  "error": "No pending registration found | Pending registration does not match provided address. | Name already registered | No valid payment transaction found for this address. | Transaction already used for registration."
}

Errors

Errors are returned as JSON with an error string. 

{
  "error": "…"
}
